4. Serwisy

MailSyncService — sync(Account, cap=200)

  Client::make(account->imapConfig()) → connect()
       │
       ├── getFolders(false) → pętla po WSZYSTKICH zwykłych folderach (od 03.06; bez Wysłane/Kosz/Spam/[Gmail])
       │
       ├── first sync (per folder)?  since(now()->subMonths(N))     N = sync_window_months
       │   incremental: since(now()->subDays(3))        ← "łap ostatnie 3 dni"
       │
       ├── $spam = SpamSender::pluck()->flip()          ← załaduj listę spamu
       ├── $rules = SenderRule::all()->keyBy(from_email) ← załaduj reguły
       │
       └── foreach $messages as $message:
               uid = $message->uid
               from = addresses($message->from)[0]
               │
               Message::updateOrCreate(
                 [account_id, folder_id, uid],           ← klucz
                 [subject, from_email, body_html, body_text,
                  is_read, is_spam, in_reply_to, ...]
               )
               │
               if wasRecentlyCreated (NOWY mail):
                 ├── spam.has(LOWER(from_email))  → is_spam = true
                 └── rule = rules.get(from_email)
                       ├── rule.catalog_id           → catalog_id
                       └── rule.assigned_admin_user_id → assigned_…
               │
               if hasAttachments:
                 attachments().delete()  +  foreach att → mail_attachments.create()
                 (TYLKO metadane — treść pobierana z IMAP dopiero przy downloadAttachment)

MailAiCategorizer — AI auto-kategoryzacja

  Message::whereNull('category_id')->limit(25)->get()
       │
       ├── buildPrompt(messages, categories)
       │     "Przypisz każdemu mailowi kategorię: {lista kategorii}"
       │
       ├── HTTP POST api.openai.com/v1/chat/completions
       │   (przez AiContentRuntimeConfig::openAiKey())
       │
       └── foreach response → Message::update({category_id, categorized_by:'ai'})

6. Przepływ: odbiór maili (IMAP sync)

  [Scheduler co minutę] → php artisan mail:sync
  [Lokalnie]            → mail-sync-loop.bat (php artisan mail:sync + timeout 60s)

       MailSync@handle()
            │
            └── foreach Account where is_active=true:
                    MailSyncService::sync(account)
                          │
                          ├─ 1. set sync_status = 'syncing'
                          ├─ 2. Client::make(imapConfig())->connect()
                          ├─ 3. getFolders(false) → pętla po zwykłych folderach (od 03.06; bez Wysłane/Kosz/Spam/[Gmail])
                          ├─ 4. per folder: query.since(date)->limit(200)->get()
                          │
                          ├─ 5. $spam  = SpamSender::pluck()->flip()
                          ├─ 6. $rules = SenderRule::all()->keyBy()
                          │
                          ├─ 7. foreach message:
                          │       Message::updateOrCreate([uid]) ← pomija istniejące
                          │       if nowy → stosuj spam + reguły nadawcy
                          │       if hasAttachments → mail_attachments (metadane)
                          │
                          ├─ 8. Folder::update(last_uid, counts, last_synced_at)
                          └─ 9. Account::update(sync_status='idle', last_sync_at)

7. Przepływ: wysyłka maila (SMTP)

  [Front — sendMail()]
       │
       ├── new FormData()
       │     account_id, to, cc, subject, body, is_html
       │     attachments[] ← File objects
       │
       └── axios.POST('argo-mail/send', fd)

  [Backend — MailController@send()]
       │
       ├── validate (attachments.* max:15MB, max:20 plików)
       ├── parseEmails(to) + parseEmails(cc)
       │
       ├── $email = new Symfony\Mime\Email()
       │     ->from(new Address(account.email, account.label))
       │     ->addTo() ->addCc()
       │     ->subject() ->text() ->html()
       │     ->getHeaders()->addTextHeader('In-Reply-To', ...)  ← reply
       │     ->attachFromPath(file->getRealPath(), name, mime)  ← pliki
       │
       ├── EsmtpTransport(host, port, tls=ssl?)
       │     ->setUsername() ->setPassword()
       │     ->send($email)
       │
       │   Gmail: automatycznie zapisuje do Sent na Gmailu
       │
       └── Message::create(is_sent=true, catalog_id=sendCatalogId())
                    sendCatalogId() → SEND (root) / [label skrzynki] (sub)
                    oba firstOrCreate → auto-tworzenie przy pierwszej wysyłce

9. IMAP — szczegóły techniczne

Biblioteka
  • webklex/laravel-imap ^6.2
  • Pure PHP — bez ext-imap
  • Połączenie: Client::make(imapConfig())
  • imapConfig() buduje tablicę z danych konta
Okno pobierania
  • Pierwsze połączenie: since(now()->subMonths(N))
  • Każde kolejne: since(now()->subDays(3))
  • Limit: 200 wiadomości / run
  • updateOrCreate po uid — duplikaty pomijane
Atrybuty wiadomości
  • $m->uid — unikalny ID w skrzynce
  • $m->subject, $m->date->toDate()
  • $m->from->toArray() → Address[]
  • $m->getHTMLBody(), getTextBody()
  • $m->getFlags()->toArray() → Seen|Flagged
  • $m->getAttachments() → kolekcja
Załączniki (lazy load)
  • Przy syncu: TYLKO metadane w mail_attachments
  • Treść NIE jest w bazie danych
  • Przy downloadAttachment(): reconnect IMAP → whereUid → getAttachments()[part_index] → content
  • Oszczędność dysku dla dużych załączników

10. SMTP — szczegóły techniczne

Transport
  • Symfony\Mailer\Transport\Smtp\EsmtpTransport
  • port 465 + encryption=ssl → $tls = true
  • port 587 + STARTTLS → $tls = false
  • setUsername / setPassword z konta
Wiadomość (Symfony Mime)
  • new Address(email, label)
  • addTo(), addCc(), subject()
  • text() + html() — zawsze oba
  • In-Reply-To / References (reply)
  • attachFromPath(getRealPath(), name, mime)
Kopia wysłanego
  • Gmail: auto-zapisuje do Sent na Gmailu
  • PIM: Message::create is_sent=true
  • folder __SENT_LOCAL (nie synchronizowany)
  • catalog_id = SEND / [label skrzynki]
  • SEND + podkatalogi — auto-tworzone firstOrCreate
FormData (wysyłka z plikami)
  • Front: new FormData() + fd.append('attachments[]', file)
  • Walidacja: max 20 plików, max 15 MB / plik
  • Backend: $request->file('attachments', [])
  • $file->isValid() sprawdzany przed attach

11. SPAM — mechanizm

  [Użytkownik klika SPAM na nadawcy]
  axios.POST messages/{id}/spam
       │
       ├── SpamSender::firstOrCreate({from_email: LOWER(email)})
       └── Message::whereRaw('LOWER(from_email)=?')->update({is_spam:true})
           ← WSZYSTKIE dotychczasowe maile tego nadawcy

  [Nowy mail przy synchronizacji]
  MailSyncService::sync()
       │
       ├── $spam = SpamSender::pluck('from_email')->map(LOWER)->flip()
       └── foreach message:
               is_spam = $spam->has(LOWER(from_email))

  [Filtrowanie w index()]
       │
       ├── filters.spam = true  → WHERE is_spam=1  (widok Spam)
       └── filters.spam = false → WHERE is_spam=0  (wszędzie indziej)

       Liczniki (accUnread, catUnread, itd.) → AND is_spam=0  ← spam nie wchodzi do liczników

  [Przywróć z listy spamu]
  axios.POST messages/{id}/unspam
       ├── SpamSender::where(from_email)->delete()
       └── Message::whereRaw('LOWER(from_email)=?')->update({is_spam:false})

12. Drag & drop — mechanizm

  [FRONT]

  <li draggable="true"
      @dragstart="onDragStart(e, m)">
       │
       └── dragMessage.value = {id, from}
           e.dataTransfer.setData('text/plain', m.id)
           dragHint.show = true  ("Upuść na katalog → przypisze nadawcę")

  <button  ← katalog
      @dragover.prevent="onCatalogDragOver(c.id)"
      @dragleave="onCatalogDragLeave(c.id)"
      @drop="onCatalogDrop(c.id)">           ← c.id = number
  <button  ← "Wszystkie"
      @drop="onCatalogDrop(null)">            ← null = cofnięcie

  [BACKEND] POST argo-mail/messages/{id}/file-sender
  {catalog_id: number | null}

  catalog_id != null  (upuścił na konkretny katalog):
       SenderRule::updateOrCreate(
         {from_email},
         {catalog_id}
       )
       Message::whereRaw('LOWER(from_email)=?')
               ->update({catalog_id})      ← WSZYSTKIE maile nadawcy

  catalog_id = null   (upuścił na "Wszystkie"):
       rule = SenderRule::where(from_email)->first()
       if rule && rule.assigned_admin_user_id:
           rule.update({catalog_id: null}) ← zostaw osobę, zdejmij tylko katalog
       else:
           rule.delete()                  ← kasuj całą regułę
       Message::whereRaw('LOWER(from_email)=?')
               ->update({catalog_id: null})

13. Filtr po kolorach — mechanizm

  [Backend — colorCounts]

  $colorCounts = (clone $query)       ← KLON query (ten sam zakres: konto/katalog/itd.)
      ->reorder()                     ← usuń ORDER BY (GROUP BY nie lubi ORDER)
      ->whereNotNull('color_flag')
      ->selectRaw('color_flag, COUNT(*) as c')
      ->groupBy('color_flag')
      ->pluck('c', 'color_flag')
  → {red:5, green:3, blue:3, orange:3}

  [Backend — filtr koloru vs "nieprzeczytane"]

  if (filters.unread && !filters.color):
      $query->where('is_read', false)     ← unread TYLKO gdy NIE ma filtru koloru

  if (filters.color):
      $query->where('color_flag', filters.color)
      ← kolor > unread (pokazuj też przeczytane — bo maile kolorujesz po otwarciu)

  [Front — selectColor(color)]

  const next = filters.color === color ? null : color   ← toggle
  if (next) localFilters.unread = false                 ← zdejmij "tylko nieprzeczytane"
  navigate({color: next})

  [Front — po setColor()]
  router.reload({only:['colorCounts']})    ← tylko kwadraciki, bez przeładowania listy

14. Bezpieczeństwo

Hasła i tokeny
  • cast 'password' => 'encrypted' (AES-256, APP_KEY)
  • $hidden = ['password', 'oauth_token'] → nigdy nie wychodzą do front-endu
  • App Password (Gmail 2FA) — spacje usuwane automatycznie przy zapisie
  • OAuth2 — planowane (faza 3)
Treść maili (XSS)
  • Renderowana w <iframe :srcdoc="bodyDoc">
  • sandbox="allow-popups allow-popups-to-escape-sandbox"
  • Brak allow-scripts → JS z maila zablokowany
  • Prepended: <base target="_blank"> → linki w nowej karcie
  • referrerpolicy="no-referrer"
Załączniki
  • Treść NIE jest składowana na dysku PIM
  • Pobierana z IMAP live przy downloadAttachment()
  • abort_unless(attachment.message_id === message.id)
  • Content-Disposition: attachment (nie inline)
Ogólne
  • Brak serwera pocztowego = mniejsza powierzchnia ataku
  • Poczta zostaje na Gmail — PIM jest tylko klientem
  • Wszystkie endpointy pod middleware: web + auth + crafter.verified
  • CSRF: Inertia + Axios X-XSRF-Token
