# 05 — UI w crafterze

## Wpisy w menu (Sidebar)
`resources/js/crafter/Components/Sidebar.vue` — pod „AI Tools":
- **Matryca tłumaczeń** (`LanguageIcon`) → `/admin/translation-phrases`
- **Tłumaczenia: review queue** (`ClipboardDocumentCheckIcon`) → `/admin/translation-review`

---

## Matryca tłumaczeń

### Lista — `/admin/translation-phrases`
`TranslationPhrase/Index.vue` · `TranslationPhraseController@index`

- Tabela 537 fraz: ID / PL fraza (+slug) / liczba produktów / liczba kanałów (kolorowy tag)
- Sort: domyślnie `-product_count` (najczęstsze na górze)
- Search: po `slug` / `phrase_pl` (FuzzyFilter)
- Tag kanałów: zielony ≥10, żółty ≥6, czerwony <6 (z 11)

### Edycja — `/admin/translation-phrases/{id}/edit`
`TranslationPhrase/Edit.vue` · `TranslationPhraseController@edit/@update`

- Pole „Klucz PL" (`phrase_pl`)
- 11 pól tekstowych (po jednym na kanał); puste = czerwone tło
- Sygnał niespójności: „X wariantów w arkuszu" (z `variants_count`)
- **Zapis** → rendition source=`manual`
- **Button „Reaplikuj do N produktów"** → `@reapply`:
  uruchamia sklejacz dla wszystkich produktów których PL matchuje tę frazę
  (pomija sloty zablokowane). Pyta o potwierdzenie.

---

## Review queue (human-in-the-loop)

### `/admin/translation-review`
`TranslationReview/Index.vue` · `TranslationReviewController@index`

Pokazuje produkty wymagające uwagi — kryterium:
- `enabled = false` (świeży import z Sumpguard), LUB
- brak wpisu `translation_overrides` dla `name` (wciąż surowy fallback)

Kolumny: ID / PL nazwa (+kod) / DE / pokrycie (X/6 locale, kolorowy tag) / status / akcje.

### Akcje per wiersz
| Button | Route | Co robi |
|---|---|---|
| **Auto-tłumacz** | `@autoTranslate` | `ProductTranslationComposer::apply()` — wypełnia z matrycy |
| **Zatwierdź** | `@approve` | `enabled=true` + `overrides.enabled='1'` we WSZYSTKICH IntegrationProduct → wpuszcza do delta-sync |
| ✏️ (edycja) | `crafter.products.edit` | Pełna karta produktu |

`@approve` używa `$suppressObserver` żeby zmiana `enabled` nie ruszyła override tłumaczeń.

## Routy
`routes/crafter.php`, sekcja „Translation matrix" (middleware `crafter.base`, prefix `admin`):

```
GET    admin/translation-phrases                      .index
GET    admin/translation-phrases/{id}/edit            .edit
PUT    admin/translation-phrases/{id}                  .update
POST   admin/translation-phrases/{id}/reapply          .reapply
GET    admin/translation-review                        .index
POST   admin/translation-review/{product}/auto-translate .auto-translate
POST   admin/translation-review/{product}/approve        .approve
```

## Build front-u (Vite)
Node jest w Laragonie, ale `npm` nie jest w PATH. Build:

```powershell
$env:Path = "C:\laragon\bin\nodejs\node-v18;" + $env:Path
cd D:\laragon\www\PIM
node node_modules\vite\bin\vite.js build
```
Weryfikacja, że strony się skompilowały:
```bash
grep -oE "TranslationPhrase/(Index|Edit)|TranslationReview/Index" public/build/manifest.json | sort -u
```
